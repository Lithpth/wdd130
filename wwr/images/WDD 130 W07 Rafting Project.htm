W07 White Water Rafting Site Project

Overview

Complete the rafting site project.

Instructions

  1. Review the Rafting Site Project Description.
  2. Review the Web Development Standards Checklist and compare each of your pages against the standards.
  3. Review and apply any relevant feedback that you have received from the previous weeks and from the self assessments.
  4. Continue to work on your rafting project to completion. Your site should have four linked pages:
    1. Home - index.html
    2. About Us - about.html
    3. Trips - trips.html
    4. Contact Us - contact.html

Testing

  1. Review the Web Development Standards Checklist and compare each of your pages against the standards.
  2. Validate your HTML and CSS using the Web Developer browser extension.
  3. Validate accessibility using the Web Developer browser extension.
  4. Run the CSS Overview tool in Dev Tools to test color usage, color contrast, font usage, and unused CSS.
  5. In a private browser window, load your site and use Dev Tools Lighthouse to run a report for Desktop in the performance, accessibility, best practices, and SEO categories. Review the notes provided.

    NOTE: The Lighthouse report will not be graded. It is provided as a tool to help you identify areas of improvement. Do not worry about fixing all the issues, just those you recognize.

  6. Fix any errors that can be identified.

Submission

  1. Commit your changes and push them to your wdd130 GitHub Pages repository.
  2. Submit the your rafting project URL in I-Learn. 
    https://githubusername.github.io/wdd130/wwr/
